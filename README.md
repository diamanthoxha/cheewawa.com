# Cheewawa.com

Source snapshot of [cheewawa.com](https://cheewawa.com), a chihuahua owner's library: health, training, breed, and care guides written by a small editorial team, with every article carrying verified citations to veterinary and welfare organizations (AVMA, AKC, ASPCA, Merck Veterinary Manual, VCA, AVSAB, and peers).

The site runs on **ChiLove**, a small custom PHP CMS built for this project.

## Repository layout

```
chilove/                  The ChiLove application (engine + theme + public docroot)
  core/                   Router, database layer, hooks, template helpers, JSON-LD
  content/themes/chilove/ The Cheewawa theme (templates, sidebar, static pages)
  content/plugins/        Newsletter + contact form plugins
  public/                 Web root: front controller, assets (css/js/img/fonts)
  migrations/             SQL schema migrations
  seed.php                Dev-database seeder
  config.example.php      Copy to config.php and fill in your values
data/                     Sanitized content snapshot (posts, terms, authors)
tools/                    gen_img_variants.sh — builds the 160/480/768/1024w srcset variants
ops/                      Cache-header rules and pending redirect blocks
```

## Running locally

1. PHP 8.x + MariaDB/MySQL. Point a vhost (or `php -S` with `public/router.php`) at `chilove/public/`.
2. `cp chilove/config.example.php chilove/config.php` and fill in DB credentials.
3. Apply `chilove/migrations/*.sql` in order, then either run `seed.php` for dev fixtures or import `data/content-snapshot-*.sql` for the real article content.
4. Image `srcset` variants (`*-480w.webp` etc.) and `*-og.jpg` social images are generated artifacts, not tracked here — run `tools/gen_img_variants.sh` over `public/assets/img/posts/` to rebuild the variants.

## Production notes

- Live deployment serves `chilove/public/` behind Apache; long-lived immutable caching for static assets per `ops/htaccess-cache.conf`.
- The production `.htaccess` additionally carries several hundred `301` rules mapping legacy WordPress-era URLs onto current `/post/{slug}` URLs (managed operationally, not in this repo). `ops/redirects-pending-cheewawa.conf` holds the most recent batch in plain `RewriteRule` form.
- JSON-LD (Article / Breadcrumb / FAQPage), Open Graph tags, canonicals, and the sitemap are generated by the engine — nothing is hand-embedded in content.
- `config.php` (real credentials, GA4 measurement ID) is intentionally untracked.

## Content

Article content lives in the database. `data/content-snapshot-*.sql` is a sanitized dump of the four content tables (`chi_posts`, `chi_terms`, `chi_post_terms`, `chi_users`) as of the snapshot date — no subscriber or contact-form data is included. Editorial canon: real facts only, no invented sources or quotes; health topics carry cautious wording and verified outbound citations.
